## Q1. 갈 수 있는 학교 입결 높은 순 5개 (학과 포함)

### 질문 해석
"입결 높은 순" = 가장 선발 기준이 높은(합격하기 어려운) 학교를 앞에. "갈 수 있는" = cut_70 기준으로 학생 등급보다 높거나 같은 학교.

### 학생 수준 분석
- 국수영 평균: (3+2+1)/3 = **2.0등급**
- 탐구 포함 평균: (3+2+1+4+5)/5 = **3.0등급**
- 수시 학생부교과 기준으로 영어 1등급이 강점, 과학 5등급이 약점
- 고1 모의고사이므로 실제 수능/내신 예측력은 낮음

### MCP 서버 응답 (match_by_grade grade=3.0, limit=5, score_type="등급")

```
1. 상명대학교 교육학과          | 학생부종합(상명인재전형) | cut_70=2.80 → 추천
2. 인천대학교 미디어커뮤니케이션  | 학생부교과(지역균형)    | cut_70=2.80 → 추천
3. 가천대학교 바이오나노학과      | 지역균형               | cut_70=2.81 → 추천
4. 전남대학교 생물공학과          | 학생부교과(일반전형)    | cut_70=2.81 → 추천
5. 가천대학교 간호학과            | 가천바람개비           | cut_70=2.82 → 추천
```

### 실제 데이터 교차 검증 (`data/results_extracted/`)

### MCP 서버 응답 가능 범위

현재 MCP 서버는 이 질문에 **직접 답변 불가**. `match_by_grade`와 `list_universities`로 데이터는 가져올 수 있지만, **전략적 조언**은 다음 정보가 없어 불가능:

3. 각 전형별 내신/수능 반영 비율

- 정시: 영어1+수학2 강점 → 수능 표준점수 280~300 목표 시 인서울 중하위권 가능

## Q6. ㅇㅇ 학과 전국, 합격률 포함 입결 순 전체

### 예시: 컴퓨터공학과 전국

**MCP `search_programs(["컴퓨터공학"], limit=50)`**: 전국 컴퓨터공학 관련 학과 조회

**실제 DB 쿼리 결과** (결과 있는 학교만):

```
수시 등급 입결 (높은 순):
1. 가천대학교 컴퓨터공학전공     | 학생부우수자          | cut_70=2.53  [tier 4, 경기]
2. 전남대학교 전자컴퓨터공학부   | 학생부교과(지역인재)  | cut_70=2.65  [tier 2, 광주]
3. 전남대학교 전자컴퓨터공학부   | 학생부교과(일반전형)  | cut_70=2.94  [tier 2, 광주]
4. 한국외국어대학교 컴퓨터공학부 | 학생부종합전형        | cut_70=3.00  [tier 3, 서울]
5. 동의대학교 컴퓨터공학과       | 일반고교과전형        | cut_70=3.16  [tier 5, 부산]
6. 명지대학교 컴퓨터공학전공     | 명지인재서류          | cut_70=3.24  [tier 4, 서울]
7. 가천대학교 컴퓨터공학전공     | 지역균형              | cut_70=3.29  [tier 4, 경기]
8. 세종대학교 컴퓨터공학과       | 특별전형              | cut_70=3.61  [tier 3, 서울]
9. 건국대학교(글로컬) 컴퓨터공학 | 학생부(교과)전형      | cut_70=4.20  [tier 4, 충북]
10. 전남대학교 컴퓨터공학전공    | 학생부교과(일반전형)  | cut_70=6.76  [tier 2, 광주]
```

**큰 문제**: 건국대학교(서울) 컴공, 중앙대, 경희대, 숭실대, 세종대 일반전형, 한양대, 성균관대 컴공 등 **핵심 학교들이 누락**.

**학생 기준 합격 가능성**:
- 한국외대 컴공 (cut_70=3.00): grade=3.0이면 margin=0 → 추천 경계
- 명지대 컴공 (cut_70=3.24): margin=+0.24 → 추천
- 가천대 컴공 지역균형 (cut_70=3.29): margin=+0.29 → 추천
- 세종대 컴공 특별전형 (cut_70=3.61): margin=+0.61 → 안정

→ 학생 모고 3등급 기준으로 **가천대/명지대/한국외대 컴공 등이 목표권**, 건국대/세종대/한양ERICA 이상은 도전 이상

### 정확도 분석

| 문제점 | 심각도 |
|--------|--------|
| 전국 컴퓨터공학 학교 중 결과 있는 곳이 10개 미만 (실제 30+ 학교) | 🔴 Critical |
| 주요 인서울 컴공 (건국/중앙/경희/숭실/한양ERICA) 완전 누락 | 🔴 Critical |
| 학과명 매칭이 "컴퓨터공학"만 됨 — "컴퓨터정보", "IT", "AI" 등 유사 학과 일부 누락 | 🟡 High |
| 합격률이 verdict(안정/추천/도전)만 나오고 % 수치 없음 | 🟠 Medium |
| 지역인재 전형 필터 없음 (지방 학생이 아닌 경우 지역인재 지원 불가) | 🟠 Medium |

## 전체 정확도 종합 평가

### 시스템이 잘 하는 것 ✅
1. **입결 데이터가 있는 학교**에 대해서는 cut_70/cut_50 값을 정확하게 반환
2. 건국대, 단국대, 동의대, 순천향대, 경북대 등 **CDN 보유 학교** 데이터 정확
3. 안정/추천/도전 **verdict 분류 로직**은 합리적
4. 대학 **tier/region** 필터링 정상 작동
5. FTS5 **전문 검색** 기능 정상 작동

### 시스템이 못 하는 것 ❌

| 기능 | 현황 | 이유 |
|------|------|------|
| 인서울 주요대 입결 조회 | ❌ | 서울대/고려대/서강대/한양대/경희대 등 CDN에 없음 |
| 모의고사 → 수능 예측 | ❌ | 두 지표의 상관관계 데이터 없음 |
| 내신 vs 수능 전형 분리 | ❌ | 두 등급을 같은 파이프로 처리 |
| 수능최저학력기준 필터 | ❌ | 모집요강 텍스트 파싱 안 됨 |
| 합격 확률 % 수치 | ❌ | 통계 모델 없음 |
| 과목별 등급 반영 비율 | ❌ | 전형별 가중치 없음 |
| 충원 합격률 | ❌ | 충원 인원 파싱 안 됨 |
| 정시 주요대 수능 컷 | ❌ | 표준점수 데이터 극히 부족 |
| 전략 조언 | ❌ | 순수 데이터 서버, 추론 기능 없음 |

## 개선 방향 (우선순위 순)

### 🔴 Critical — 즉시 개선 필요

#### 1. 주요 대학 입시결과 보강 (가장 중요)
- **현황**: 113개 대학 (62%) 입시결과 없음. 서울대, 고려대, 연세대, 한양대, 서강대, 경희대 등 인서울 주요 대학 대부분 CDN 없음.
- **방안 A**: 각 대학 홈페이지 직접 크롤링 (경희대입학처: https://iphak.khu.ac.kr, 한양대: https://www.hanyang.ac.kr/admission)
- **방안 B**: 어디가(어디가어디가.com) API 또는 adiga.kr 입시결과 PDF 활용
- **방안 C**: 대학알리미(academyinfo.go.kr) 공공데이터 활용

#### 2. 모의고사 vs 내신 등급 분리
- **현황**: `match_by_grade(grade=3.0)`이 수시 교과 컷에 직접 대입됨
- **방안**: `score_type` 파라미터에 `"내신"` / `"모의고사"` 구분 추가
  - 내신 등급 → 수시 학생부교과/종합 컷과 비교
  - 모의고사 등급 → 수능 예측 후 정시 표준점수 범위로 변환하는 별도 함수 필요
- **구현 아이디어**: 등급 → 표준점수 변환 테이블 추가 (`등급_표준점수_변환.json`)

#### 3. 수능최저학력기준 파싱
- **현황**: 모집요강 content에 수능최저 조건이 텍스트로 있으나 파싱 안 됨
- **방안**: `admission_process.attributes`에 `수능최저` 필드 추가
  - 파싱 패턴 예시: "수능 2개 합 5이내", "국수영탐 중 2개 2등급 이상"
  - 정규식으로 추출 → `attributes["수능최저"]` 저장

#### 4. (전형미상) 전형명 해결
- **현황**: 1,891건 (13.6%)이 전형미상 — 중앙대 110건 전체, 서울시립대 등 주요대 다수
- **방안**:
  - 페이지 헤더 텍스트 기반 파싱 재시도 (현재 패턴이 누락하는 케이스 분석)
  - `find_process_context_from_text()` 개선 — 표 이전 텍스트 더 많이 탐색

---

### 🟡 High — 중기 개선

#### 5. 정시 표준점수 데이터 확충
- **현황**: 표준점수 컷 1,528건이지만 대부분 지방 국립대, 인서울 정시 거의 없음
- **방안**:
  - 진학사/유웨이 등에서 공개하는 정시 합격 선 데이터 수집
  - 각 대학 입학처 공개 자료 활용

#### 6. 충원 합격률 파싱
- **현황**: 실제 파일(예: 건국대 컴공 충원 76명)에 충원 인원 있으나 DB 미저장
- **방안**: `admission_result.attributes["충원인원"]` 저장 → 실질 경쟁률 = 모집인원÷(초기합격+충원)

#### 7. 계열(인문/자연) 필터 개선
- **현황**: `track` 컬럼 값이 불규칙 ("자연 과학", "공학", "자연", "이공계" 등 혼재)
- **방안**: 학과명 기반 계열 분류 함수 추가 (키워드 기반)

#### 8. 과목별 등급 입력 → 전형별 가중치 반영
- **현황**: 단일 등급 하나만 입력
- **방안**: `match_by_grade`를 확장하여 `{"국어": 3, "수학": 2, "영어": 1, "탐구": 4.5}` 형태 입력 지원
  - 학생부교과 전형: (국+수+영) 또는 (국+수+영+탐구) 가중치 합산
  - 수능 정시: 수능 표준점수 환산

#### 9. 지역인재 전형 태깅
- **현황**: 지역인재 전형(경북대 교과지역, 전남대 지역인재 등)이 일반 전형과 혼재
- **방안**: `attributes["지역인재"] = true/false` 파싱 추가

---

### 🟠 Medium — 장기 개선

#### 10. 합격 확률 수치화
- **방안**: `(grade - cut_70) / (cut_90 - cut_50)` 형태의 정규화 점수 계산 → 0~100% 범위로 변환
- cut_50=1.64, cut_70=1.75, grade=1.75일 때 → "70% 합격 선에 해당"으로 표현

#### 11. 경쟁률 기반 지원 전략 필터
- **방안**: 경쟁률 낮은 전형 우선 추천 (`competition_rate ASC` 옵션 추가)

#### 12. 전년도 데이터 트렌드 비교
- **현황**: 2024년/2025년 결과가 혼재되어 있음
- **방안**: 연도별 cut_70 변화 추이 계산 → "매년 0.1등급씩 하락 중" 등 표현

#### 13. 모집요강 수능최저 + 입결 연동
- 수능최저 조건과 실제 합격 컷을 함께 표시: "수능최저 만족 시 실질 경쟁률은 4.7:1"

#### 14. 유사 학과 확장 매핑
- "컴퓨터공학" 검색 시 "소프트웨어학부", "IT공학", "정보통신공학", "컴퓨터정보학" 등도 포함
- 현재 `CATEGORY_KEYWORDS` 확장으로 부분적으로 가능하나 완전하지 않음

## 데이터 품질 이슈 목록

| 이슈 | 발생 규모 | 영향 |
|------|-----------|------|
| 학과명에 공백 삽입 ("정 보 통 신 공 학 과") | 다수 | 검색 누락 |
| (전형미상) 전형명 | 1,891건 (13.6%) | 전형 필터링 불가 |
| cut_50 없는 레코드 | 다수 | 도전 판정 불가 |
| 경쟁률 없는 레코드 | 상당수 | 실질 경쟁률 표시 불가 |
| 입력 기준 불명확 (모집인원 3명 이하 비공개 — 건국대 예) | 일부 | 소수 선발 학과 누락 |
| result_year 2021-2023 데이터 포함 (오래된 데이터) | 일부 | 최신성 이슈 |
| 건국대 컴공 admission_type이 "정시"로 잘못 저장 | 확인됨 | 수시/정시 필터 오류 |

**건국대 컴공 admission_type 오류 상세**:
```sql
-- DB: admission_type='정시'인데 실제는 수시 학생부교과전형
SELECT admission_type, process_name FROM admission_result r
JOIN admission_department d ON d.id = r.department_id
WHERE d.university='건국대학교' AND d.name LIKE '%컴퓨터%'
-- 결과: '정시' (실제 파일에는 "Ⅰ 2024학년도 전형결과 ― 수시 학생부교과전형"으로 명시)
```

→ `extract_results_batch.py`에서 `admission_type` 기본값 또는 파싱 로직에 버그 있음

## 결론

현재 MCP 서버는 **보유한 데이터 범위 내에서는 입결 조회가 정확**하지만, 실용적인 입시 상담 도구로서는 다음 이유로 심각한 한계가 있다:

1. **데이터 커버리지 38%** — 가장 궁금한 인서울 주요대 대부분 없음
2. **모의고사-내신-수능 등급 혼용** — 근본적인 개념 오류
3. **수능최저 조건 없음** — 실제 합격 가능성 판단 불가
4. **전형미상 13.6%** — 전형 기반 필터링 신뢰도 낮음

**6개 질문 중 제대로 답할 수 있는 것**: Q6 컴퓨터공학 전국 입결 부분 조회 (보유 데이터 범위 내), Q4 건국대 컴공 기초 컷 확인 — **2/6 질문만 부분적으로 유용**한 답변 가능.

나머지 질문들은 **LLM이 이 MCP를 보조 도구로 활용**하여 데이터 조회 + 추론을 결합할 때만 의미 있는 답변이 가능하며, LLM 없이 MCP 단독으로는 실용적 입시 상담 불가.
